# CLAUDE.md

Guidance for agents working in this repository. Read this before making changes.

## What this is

AutoSuggest is a system-wide macOS autocomplete: it watches your keystrokes in
any app and offers inline completions generated by **local** LLMs (Ollama,
llama.cpp, or Core ML). It is a SwiftPM package (library `AutoSuggestApp` +
executable `AutoSuggestRunner`) plus an xcodegen-generated Xcode app shell in
`macos/` (`macos/project.yml`).

## Build / test / lint

```sh
swift build                            # exit 0 on success (~1 min cold, seconds warm)
swift test                             # full suite (158 tests, ~70s on Apple Silicon)
swiftformat Sources Tests --lint       # formatter gate — must report 0 files require formatting
cd macos && xcodegen generate          # regenerate the Xcode app project from project.yml
```

> Note: `.swiftformat` (repo root) holds the canonical config. Format in place
> with `swiftformat Sources Tests`. With SwiftFormat 0.61.x the lint invocation
> is `swiftformat Sources Tests --lint` (paths before the flag); CI uses the
> `--lint` form against the installed brew version.

## Architecture map

The keystroke-to-insertion pipeline, one stage per line with its source file:

1. `Sources/AutoSuggestApp/Input/CGEventInputMonitor.swift` — taps CGEvents to observe keystrokes.
2. `Sources/AutoSuggestApp/Context/AXTextContextProvider.swift` — reads surrounding text via the Accessibility API.
3. `Sources/AutoSuggestApp/System/PolicyEngine.swift` — decides whether suggesting is allowed here (app/field policy, secure-field suppression).
4. `Sources/AutoSuggestApp/Suggestions/SuggestionOrchestrator.swift` — debounces input (~150ms) and schedules requests, cancelling stale ones.
5. `Sources/AutoSuggestApp/Inference/InferenceEngine.swift` — runs the runtime fallback chain (ollama → llama.cpp → coreml) per `config.json` `runtimeOrder`.
6. `Sources/AutoSuggestApp/Suggestions/FloatingOverlayRenderer.swift` — draws the floating completion overlay.
7. `Sources/AutoSuggestApp/Suggestions/AXTextInsertionEngine.swift` — inserts the accepted completion (paste-first insertion).

`Sources/AutoSuggestApp/Suggestions/TypingPipeline.swift` wires these stages
together. Config lives at `~/Library/Application Support/AutoSuggestApp/config.json`,
with versioned migrations in `Sources/AutoSuggestApp/Config/ConfigMigrationManager.swift`.

## Critical paths — extra care

- **Text insertion** (`Sources/AutoSuggestApp/Suggestions/AXTextInsertionEngine.swift`):
  inserts into the user's live document and can corrupt their text. It is
  paste-first and must preserve/restore the clipboard. Lean on the
  characterization tests before touching it.
- **Policy / secure-field suppression** (`Sources/AutoSuggestApp/System/PolicyEngine.swift`,
  `Sources/AutoSuggestApp/Context/AXTextContextProvider.swift`): this is the
  password-safety boundary. Suggestions must never fire in secure (password)
  fields. Do not weaken these guards.
- **Config migrations** (`Sources/AutoSuggestApp/Config/ConfigMigrationManager.swift`):
  must never break existing v0/v1 configs. Any user-facing config schema change
  needs a migration step here.
- **Privacy invariants**: never log or persist raw typed text or completions.
  PII-filter before any persistence (`Sources/AutoSuggestApp/Privacy/PIIFilter.swift`);
  the personalization store is encrypted (`Sources/AutoSuggestApp/Privacy/EncryptedFileStore.swift`).
  Telemetry is off by default and stays content-free (no typed text, no
  completions in any event).

## Conventions

- Tests use XCTest with `@testable import AutoSuggestApp`. Test files mirror
  source names (a `PolicyEngine` type → `Tests/AutoSuggestAppTests/PolicyEngineTests.swift`).
- Log via the `Logger(scope:)` wrapper (`Sources/AutoSuggestApp/Support/Logger.swift`),
  never `print`.
- Any user-facing config change requires a corresponding `ConfigMigrationManager` step.
- UI strings are currently hardcoded English (no localization layer yet).

## Gotchas

- Real AX/CGEvent behavior cannot be exercised in CI; those paths are exercised
  through mocks in `Tests/AutoSuggestAppTests/IntegrationTestHarness.swift`.
- Permission-dependent behavior (Accessibility/Input Monitoring grants) must be
  tested through the Xcode app target (`macos/`), not the SwiftPM
  `AutoSuggestRunner`.
