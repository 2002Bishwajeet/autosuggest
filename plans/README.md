## Findings considered and rejected

Recorded so they aren't re-audited:

- **Stale-context insertion race (TypingPipeline)**: already guarded — `handleShortcut` re-fetches and validates context before inserting (`TypingPipeline.swift:147-151`).
- **`latestRequestID` race (SuggestionOrchestrator)**: `@MainActor` serializes scheduling; `Task.isCancelled` + requestID double-check cover the await window. Fragile-looking but correct.
- **Overlay "stale panel pointer crash"**: panel is captured strongly in the hide completion — no crash. The real (milder) issue is the hide/show animation race, fixed in plan 005.
- **"Cache AX context between keystrokes" (perf)**: would break suggestion-validity checks, which need fresh context per keystroke. Per-keystroke AX reads are inherent to the design.
- **install.sh MITM hardening**: GitHub-over-TLS is the standard installer trust root; notarization (plan 008) is the real binary-trust fix. Follow-up noted in plan 008's maintenance notes (`codesign --verify` in install.sh).
- **Telemetry schema versioning**: premature — telemetry is local-only, off by default, and content-free after plan 002.
- **User-configurable manifest URL as SSRF-ish risk**: by-design for a local tool operating on the user's own config; integrity enforcement (plan 006) covers the meaningful part.
- **Fire-and-forget telemetry Task accumulation**: actor-serialized, negligible volume; hygiene not worth a plan.
- **llama.cpp error-handling asymmetry** (`LlamaCppInferenceRuntime.swift:99` vs `:72-75`): `InferenceEngine` catches all thrown errors and continues the fallback chain, so impact is minor; folded conceptually into the (deferred) adapter consolidation below.
- **AppCoordinator/AutoSuggestViews/OnboardingFlowView god objects (~920 lines each) + 3 duplicated HTTP adapters**: real debt, HIGH refactor risk, not launch-relevant — deliberately deferred until after v0.1.0 + tests mature. Revisit when a feature forces a change in those files.
- **Hardcoded UI strings bypassing `Localizable.strings`**: real, low urgency for a single-locale launch; bundle with a future localization effort.
- **Configurable accept/dismiss shortcuts** (Tab/Return hardcoded, `CGEventShortcutMonitor.swift:11-12`): user chose not to plan it now; the Return-as-accept hazard in chat apps is worth revisiting from user feedback after launch.
- **In-app fine-tune loop**: training scripts + export exist (`training/`, `Privacy/TrainingDataExporter.swift`); full in-app loop is premature pre-launch. Cheap interim: document the export → finetune → install path in the README (no plan needed).
- **Onboarding doesn't re-run on permission revocation**: partially mitigated — pause reason already surfaces permission problems in the popover (`AppCoordinator.swift:826-828`), and plan 005 adds the remedy text. A dedicated repair flow is deferred.
